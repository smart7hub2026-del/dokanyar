# استقرار production (دکان‌یار)

## متغیرهای مهم محیط

| متغیر | توضیح |
|--------|--------|
| `NODE_ENV=production` | فعال‌سازی CORS محدود، کوکی امن، الزام HTTPS در صورت وجود پروکسی |
| `JWT_SECRET` | الزامی؛ رشته طولانی تصادفی |
| `ALLOWED_ORIGINS` | لیست مبدأهای مجاز جدا با ویرگول، مثلاً `https://app.example.com` |
| `ALLOW_TRIAL_QUICK_SIGNUP` | پیش‌فرض در production: **خاموش**. برای فعال کردن ثبت‌نام آزمایشی سریع (موبایل+رمز) مقدار `true` بگذارید |
| `AUTO_BACKUP_INTERVAL_HOURS` | اگر بزرگ‌تر از ۰ باشد، کپی خودکار SQLite در `server/backups` هر N ساعت |
| `DATABASE_URL` | پیش‌فرض `file:./server/prisma/dev.db` — در production مسیر مطلق فایل دیتابیس توصیه می‌شود |

## HTTPS

پشت **reverse proxy** (nginx، Caddy، Cloudflare) هدر `X-Forwarded-Proto: https` را ارسال کنید؛ سرور در production درخواست‌های غیر HTTPS را با ۴۲۶ رد می‌کند.

## پشتیبان و بازیابی

- از پنل **ابرادمین → داشبورد → عملیات پلتفرم → پشتیبان پایگاه داده** یا `POST /api/master/platform-backup` (با نشست ابرادمین) استفاده کنید.
- برای بازیابی، سرویس را متوقف کرده، فایل `.db` پشتیبان را جایگزین فایل فعلی کنید و دوباره اجرا کنید.

## نسخهٔ کلاینت و state

همگام‌سازی فروشگاه فیلد `_schemaVersion` را در payload state ارسال می‌کند؛ برای مهاجرت‌های آینده در سرور یا کلاینت قابل استفاده است.

## مانیتورینگ سبک

`GET /api/health` — بدون احراز هویت؛ مناسب چک لایو بودن سرویس (خارج از محدودیت نرخ سنگین اولیه).
